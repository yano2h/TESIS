\chapter{Descripción de las clases}
\label{anexo4}

Para una mayor comprensión del diagrama de clases, se presenta una breve 
	explicación de cada clase y los atributos y/o métodos que la componen.

	\begin{enumerate}

		\item \textbf{Clase Funcionario:} Es la clase que representa a los 
		usuarios del sistema, principalmente a los que se dedican a enviar
		solicitudes de requerimiento a DISICO.\\\\
		\textbf{Atributos de la Clase:}
		\begin{itemize}
			\item \textbf{rut:} Es el rut de la persona en formato entero sin 
			puntos guiones ni dígito verificador, que sirve para identificar a un 
			usuario de manera unequivoca.
			\item \textbf{nombre:} Cadena que contiene el nombre o nombres 
			del funcionario.
			\item \textbf{apellidoPaterno:} Cadena que almacena el apellido 
			paterno (o primer apellido) del funcionario.
			\item \textbf{apellidoMaterno:} Cadena que almacena el apellido 
			materno (o segundo apellido) del funcionario.
			\item \textbf{correoUV:} Cadena que almacena el correo 
			institucional (con dominio @uv.cl) del funcionario.
			\item \textbf{fechaPrimerAcceso:} Fecha y hora en que accedió por
			primera vez el funcionario al sistema.
			\item \textbf{fechaUltimoAcceso:} Fecha y hora en que accedió por
			ultima vez el funcionario al sistema.
			\item \textbf{notificaciones:} Lista del tipo Notificación con todas las notificaciones
			del usuario.
		\end{itemize}
		
		\item \textbf{Clase FuncionarioDisico:} Esta clase extiende a de la 
		clase Funcionario y representa a los funcionarios de la universidad que
		trabajan en el departamento de DISICO.\\\\
		\textbf{Atributos de la Clase:}
		\begin{itemize}
			\item Hereda todos los atributos de la clase Funcionario.
			\item \textbf{cargo:} Cadena con el cargo que tiene el funcionario
			dentro de Disico.
			\item \textbf{anexo:} Cadena con el anexo si es que posee para comunicarse
			por teléfono con el funcionario.
			\item \textbf{estadisticasPersonales:} Un lista (del tipo EstadisticaPersonalDiaria) que 
			contiene todas las mediciones estadísticas del usuario que se han tomado desde que fue 
			registrado en el sistema.
			\item \textbf{area:} Área de Disico a la que pertenece el funcionario.
		\end{itemize}

		\item \textbf{Clase Area:} Esta clase representa a un área de DISICO.\\\\
		\textbf{Atributos de la Clase:}
		\begin{itemize}
			\item \textbf{codigoArea:} Entero con el código numero identificador del área.
			\item \textbf{nombreArea:} Cadena con el nombre del área.
			\item \textbf{descripcionArea:} Cadena con la descripción del área y sus funciones.
		\end{itemize}

		\item \textbf{Clase SolicitudDeRequerimiento:} Clase que representa representa a
		las solicitudes de requerimientos enviadas por los usuarios del sistema.\\\\
		\textbf{Atributos de la Clase:}
		\begin{itemize}
			\item \textbf{idSolicitud:} Entero con un numero identificador para la solicitud.
			\item \textbf{codigoConsulta:} Cadena con un código único, utilizado para que
			los usuario puedan buscar solicitudes a través de este código, a diferencia del
			idSolicitud esta cadena no es secuencial, es un conjunto de caracteres mas un dígito 
			verificador que permite corroborar la valides del código.
			\item \textbf{asunto:} Cadena con una descripción corta pero representativa de lo que trata
			la solicitud.
			\item \textbf{mensaje:} Cadena que contiene el mensaje completo explicando de que 
			consiste la solicitud. 
			\item \textbf{justificacionTransferencia:} Cadena utilizada en caso de que el Jefe de Área 
			transfiera una solicitud a otra área, para justificar el motivo de esta transferencia.
			\item \textbf{respuesta:} Cadena que contiene el mensaje de respuesta ingresado una vez 
			que se da por cerrada la solicitud.
			\item \textbf{fechaEnvio:} Fecha y hora en la que fue enviada la solicitud.
			\item \textbf{fechaCierre:} Fecha y hora en la que fue cerrada la solicitud.
			\item \textbf{fechaVencimiento:} Fecha y hora que se a definido como plazo máximo
			para que el responsable resuelva la solicitud.
			\item \textbf{fechaUltimaActualizacion:} Fecha y hora en que fue modificada por ultima vez 
			la solicitud ya sea por creación, edición, transferencia, comentario, cierre de esta.
			\item \textbf{areaResponsable:} Objeto de la clase Área que contiene la información del 
			área a la que fue enviada la solicitud.
			\item \textbf{solicitante:} Objeto de la clase Funcionario que contiene la información del
			funcionario que envió la solicitud a DISICO.
			\item \textbf{responsable:} Objeto de la clase FuncionarioDisico que contiene información 
			sobre el funcionario al que fue asignada la solicitud.
			\item \textbf{tipoSolicitud:} Objeto del tipo TipoSolicitudRequerimiento con información 
			sobre el tipo de solicitud que corresponde.
			\item \textbf{estadoSolicitud:} Objeto del tipo EstadoSolicitudRequerimiento con 
			información del estado actual de la solicitud.
			\item \textbf{prioridadSolicitud:} Objeto de la clase Prioridad con información sobre la 
			prioridad que se le a asignado a la solicitud.
			\item \textbf{comentarios:} Lista del tipo ComentarioSolicitud que contiene todos
			los comentarios que se han echo sobre la solicitud.
		\end{itemize}

		\item \textbf{Clase Notificacion:} Esta clase contiene las notificaciones que son mostradas
		en la pantalla de cada usuario de manera automática.\\\\
		\textbf{Atributos de la Clase:}
		\begin{itemize}
			\item \textbf{idNotificacion:} Entero con el numero identificador de la solicitud.
			\item \textbf{fechaNotificacion:} Fecha y hora en la que se genero la notificación.
			\item \textbf{mensajeNotificacion:} Cadena que contiene el mensaje que se muestra en
			la notificación.
			\item \textbf{revisada:} Variable booleana que indica si la notificación ya a sido revisada
			por el usuario.
		\end{itemize}

		\item \textbf{Clase EstadisticaPersonalDiaria:} Esta clase contiene mediciones de diferentes
		indicadores, los cuales son calculados de manera automática al final de cada día para cada
		funcionario de DISICO.\\\\
		\textbf{Atributos de la Clase:}
		\begin{itemize}
			\item \textbf{idEstadistica:} Entero identificador de la estadística.
			\item \textbf{fechaMedicion:} Fecha en la que se tomo la medición.
			\item \textbf{totalSolicitudesAsegnadas:} Entero con la cantidad total de solicitudes que
			han sido asignadas al funcionario, desde que se registro al funcionario en el sistema
			a la fecha de la medición.
			\item \textbf{cantidadSolicitudesPendientes:} Entero con la cantidad de solicitudes 
			que han sido asignadas al funcionario y que se encuentran pendientes (pero no vencidas)
			a la fecha de la medición. 
			\item \textbf{cantidadSolicitudesVencidas:} Entero con la cantidad de solicitudes 
			vencidas (que han excedido su fecha de vencimiento y no se encuentran cerradas) 
			a la fecha de la medición.
			\item \textbf{cantidadSolicitudesIniciadas:} Entero con la cantidad de solicitudes que
			que el funcionario mantiene en estado iniciado a la fecha de la medición (no incluye 
			solicitudes vencidas).
			\item \textbf{cantidadProyectosAcargo:} Entero con la cantidad de proyectos en los que
			el funcionario participa como jefe de proyecto.
			\item \textbf{cantidadProyectosQueParticipa:} Entero con la cantidad de proyectos
			en los que participa el funcionario con un rol distinto al de jefe de proyecto.
			\item \textbf{cantidadTareasProyecto:} Entero con la cantidad de tareas de proyecto
			que el funcionario tiene abiertas a la fecha de la medición.
			\item \textbf{cantidadTareasSCM:}  Entero con la cantidad de tareas de SCM
			que el funcionario tiene abiertas a la fecha de la medición.
		\end{itemize}

		\item \textbf{Clase ComentarioSolicitud:} Clase que almacena los comentarios que los usuarios

		\textbf{Atributos de la Clase:}

			\item \textbf{fechaComentario:} Fecha y hora en la que se realizo el comentario.
			\item \textbf{comentario:} Cadena con el contenido del comentario.
			\item \textbf{visible:} Variable booleana que indica si el comentario debe ser mostrado
			o no.
			\item \textbf{autor:} Objeto de la clase Funcionario que tiene la información del
			autor del comentario.
		\end{itemize}

		\begin{itemize}

		\item \textbf{Clase SolicitudDeCambio:}\\\\
		\textbf{Atributos de la Clase:}
		\begin{itemize}
			\item \textbf{idSolicitud:} Entero identificador de la solicitud de cambio.
			\item \textbf{fechaEnvio:} Fecha y hora en que fue enviada la solicitud de cambio.
			\item \textbf{descripcionNecesidadDelCambio:} Cadena en la que se describe el motivo
			por el cual es necesario realizar el cambio.
			\item \textbf{descripcionCambio:} Cadena con la descripción del cambio concreto que 
			debe ser implementado.
			\item \textbf{fechaAnalisis:} Fecha y hora en la que se ingreso el análisis del impacto
			al sistema.
			\item \textbf{descripcionImpactoDelCambio:} Cadena que contiene el análisis del impacto
			realizado.
			\item \textbf{fechaCierre:} Fecha y hora en la que fue cerrada la solicitud.
			\item \textbf{moduloAfectado:} Cadena con la descripción del modulo o módulos
			afectados por el cambio.
			\item \textbf{descripcionResolucion:} Cadena que contiene una descripción sobre
			el motivo de la resolución que se tomo ya sea esta aprobación e rechazo.
			\item \textbf{prioridadSolicitud:} Objeto de la clase PrioridadSolicitud que contiene la 
			información de la prioridad asignada a la solicitud de cambio.
			\item \textbf{estadoSolicitud:} Objeto de la clase EstadoSolicitudDeCambio que contiene
			la información del estado actual de la solicitud.
			\item \textbf{formularioImpl:} Objeto de la clase FormularioDeImplementacion que contiene
			la información del formulario de implementación que debe completarse en caso de 
			ser aprobada la solicitud.
			\item \textbf{solicitante:} Objeto de la clase FuncionarioDisico con los datos del funcionario
			que envió la solicitud de cambio.
			\item \textbf{evaluadorImpacto:} Objeto de la clase FuncionarioDisico con los datos del 
			funcionario que realizo la evaluación del impacto de la solicitud.
			\item \textbf{evaluadorFinal:} Objeto de la clase FuncionarioDisico que realizo la evaluación
			final de la solicitud.
		\end{itemize}

		\textbf{Atributos de la Clase:}
		\begin{itemize}
			\item \textbf{idFormulario:} Entero identificador del formulario.
			\item \textbf{observaciones:} Cadena que contiene observaciones sobre la implementación

			\item \textbf{fechaVerificacion:} Fecha en la que se debe llevar a cabo la verificación del

		\item \textbf{Clase TareaSCM:} Clase que contiene la información de una tarea de SCM que

			\item \textbf{entregablesAsociados:} Lista de la clase Entregable que contiene información
			de todos los entregables asociados a dicha tarea.

			responsable de la tarea.

		\item \textbf{Clase Proyecto:} Esta clase representa los proyectos que se realizan en DISICO.\\\\
		\textbf{Atributos de la Clase:}
		\begin{itemize}
			\item \textbf{idProyecto:} Entero con el numero identificador del proyecto.
			\item \textbf{codigoInterno:} Cadena utilizada para definir un codigo interno 
			para el proyecto que sera mostrado en pantalla en vez del id.
			\item \textbf{nombre:} Cadena con el nombre del proyecto.
			\item \textbf{descripcion:} Cadena con una descripción de los objetivos y alcances del 
			proyecto.
			\item \textbf{fechaInicio:} Fecha en la que se inicio el proyecto.
			\item \textbf{fechaTermino:} Fecha en la que se cerro el proyecto.
			\item \textbf{tipoProyecto:} Objeto de la clase TipoProyecto con información del
			tipo de proyecto.
			\item \textbf{estadoProyecto:} Objeto de la clase EstadoProyecto que contiene información
			del estado actual del proyecto.
			\item \textbf{tareasSCM:} Lista de la clase TareaSCM que contiene todas las tareas scm del
			proyecto.
			\item \textbf{participantes:} Lista de la clase Participante que contiene información de los
			participantes del proyecto y su rol dentro de este.
			\item \textbf{itemsDeConfiguracion:} Lista de la clase ItemDeConfiguracion que contiene
			información de todos los items de configuración asociados al proyecto.
			\item \textbf{tareasAgendadas:} Lista de la clase TareaProyecto con información de todas
			las tareas agendadas en el proyecto. 
		\end{itemize}

		\textbf{Atributos de la Clase:}

		en un proyecto.\\\\
		\textbf{Atributos de la Clase:}
		\begin{itemize}
			\item \textbf{idTareaProyecto:} Entero identificador de la tarea.
			\item \textbf{descripcionTarea:} Cadena con una breve descripción de la tarea.

			\item \textbf{fechaTerminoReal:} Fecha y hora en la que realmente se dio fin a la tarea.

		\item \textbf{Clase EjbSolicitudRequerimeinto:} Es una clase que implementa un Ejb con 
		métodos dedicados a la gestión de solicitudes de requerimientos.\\\\
		\textbf{Métodos de la Clase:}
		\begin{itemize}
			\item \textbf{generarCodigoConsulta:} Método que genera una cadena con un código de 
			consulta unico.
			\item \textbf{validarCodigoConsulta:} Método que verifica si un código de consulta es 
			valido.
			\item \textbf{buscarSolicitudPorCodigo:} Método para buscar solicitudes a través de un 
			código de consulta.
			\item \textbf{buscarSolicitudPorSolicitante:} Método para buscar solicitudes por solicitante.
			\item \textbf{buscarSolicitudPorResponsable:} Método para buscar solicitudes por 
			responsable.
			\item \textbf{buscarSolicitudPorArea:} Método para buscar solicitudes por área.
			\item \textbf{enviarSolicitud:} Método para enviar solicitudes a un área de DISICO.
			\item \textbf{cerrarSolicitud:} Método para cerrar una solicitud.
			\item \textbf{respuestaDirecta:} Método para enviar una respuesta directa al usuario que
			envió la solicitud.
			\item \textbf{respuestaJefeArea:} Método para derivar la responsabilidad de responder la 
			solicitud al jefe de area.
			\item \textbf{respuestaManual:} Método para enviar una respuesta manualmente.
			\item \textbf{asignarResponsable:} Método para asignar responsable a una solicitud.
		\end{itemize}

		\item \textbf{Clase EjbSSO:} Es una clase que implementa un Ejb con métodos para el manejo

		\textbf{Métodos de la Clase:}
		\begin{itemize}

			\item \textbf{cerrarSesion:} Método para cerrar una sesión de usuario.

		\item \textbf{Clase EjbGeneradorDeIndicadores:} Es una clase que implementa un Ejb 
		con métodos para el calculo de indicadores y estadísticas.\\\\
		\textbf{Métodos de la Clase:}
